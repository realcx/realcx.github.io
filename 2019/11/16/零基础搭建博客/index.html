零基础搭建博客

发表于 2019-11-16

简述

对博客的预期

  • 支持markdown
  • 无需自建数据库与后台
  • 轻便，可移植性强，模板丰富，插件丰富
  • 分享技术相关的博客文章
  • 进行个人对奇奇怪怪领域的知识整理收集

本文用到的工具

  • github
  • hexo
  • Next博客模版

Github

首先拥有github账号，创建新的仓库用来存储博客的所有内容。repository name的前缀要与Owner一致。

创建完仓库，在setting界面的Github Pages就可以查看到博客的地址了，此时博客已经建立完成。

其中Custom domain是用来链接自己的域名的，可以通过阿里云购买域名进行绑定。

此时输入网址，就已经可以看到博客了。


Hexo

hexo是一款轻量级的博客模版部署工具，通过hexo对博客进行直接管理，最后将所有内容部署在github上，外部用户再通过博客地址进行访问。


安装

安装git

安装node.js：https://nodejs.org/en/#download

安装hexo：$npm install -g hexo-cli

安装hexo-git：$npm install hexo-deployer-git --save


使用

在本地建立用来装博客模版的文件夹hexo_dir，进入文件夹后操作如下

$ hexo init

$ npm install​

$ hexo g 或 $ hexo generate

$ hexo s 或 $ hexo server

本地浏览器输入：localhost:4000

此时本地博客已经搭建完成，在本地就可以进行访问。


部署

接下来需要将本地博客部署到github上，更改文件夹下的_config.yml文件

将repository这一栏改成自己的github仓库地址，此时即完成hexo到github的链接


指令

清除本地缓存

$ hexo clean

生成静态文件，即编译博客

$ hexo generate 或 $ hexo g

本地server展示

$ hexo server 或 $ hexo s

部署到github

$ hexo deploy 或 $ hexo d


添加文章

生成新的markdown博文

$ hexo new post "YOUR ARTICLE TITLE"

会产生source/_posts/YOUR-ARTICLE-TITLE.md文件；或者可以讲已经写好的markdown直接复制到source/_posts/下，下次编译的时候会直接load

$ hexo s -g

在本地查看无误后，进行部署

$ hexo d

部署之后，通过访问你的github主页进行查看，如https://realcx.github.io/


更改主题

本文中用的是Next主题，首先进入你建立的hexo文件夹

$ cd YOUR-HEXO-DIR

$ git clone https://github.com/iissnan/hexo-theme-next themes/next

然后打开_config.yml，将theme更新成next

编译部署hexo d -g，再进入github page查看

至此，个人博客的基本框架已经完成了，接下来只需要完善细节与内容


文章内容

markdown中插入图片

更改主页配置文件_config.yml

post_asset_folder:这个选项设置为true

再下载安装一个可以上传本地图片的插件，在hexo目录下执行

npm install hexo-asset-image --save，

在source/下建立文件夹images，将图片存入，博客内可以使用绝对路径表示文件地址![](/images/IMAGE.png)
