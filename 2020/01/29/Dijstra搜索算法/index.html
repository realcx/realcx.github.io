Dijkstra搜索算法

发表于 2020-01-29 |

摘要：Dijkstra搜索算法的原理与算法实现

1. 算法简介

1.1 算法定义

Dijkstra搜索算法：”又叫狄克斯特拉算法，是从一个顶点到其余各顶点的最短路径算法，解决的是有权图中最短路径问题。主要特点是以起始点为中心向外层层扩展，直到扩展到终点为止。” — from 百度百科

1.2 要解决什么问题

  • 找到从起点A到达终点B的最短路径；
  • 在AB两点之间可能会有障碍物阻碍，可穿越，但穿越障碍物需要进行惩罚；
  • 也可以理解为，各个节点之间的移动cost是不同的，要找到有权图中的最短移动路径；

1.3 大致解决思路

类似于根据历史移动cost做出的宽度有限搜索：即从起点A开始出发，遍历周围的节点，再像洪水一样逐层地向外扩散，并且记录从起始点到途经点的移动cost，每次再以最小cost的点作为新的节点继续开始探索，直到找到终点。


2. 典型例子

基于图搜的路径规划问题

2.1 在地图中的元素与规则

  • 起点A（蓝色）
  • 终点B（红色）
  • 障碍物（灰色）
  • 正常节点之间的移动cost为1，穿越障碍物时的移动cost为5
  • 节点之间只能”上下左右”移动

2.2 定义两个队列

  • open_set：表示要去探索的点（有顺序要求），图中用绿色表示
  • close_set：表示已经探索过的点（无顺序要求），图中用黄色表示

2.3 算法过程

则地图中探索的优先顺序为：

  1. cost最低的做为新节点；
  2. 若cost相同，按照“上-右-下-左”的顺序选取新节点；
  3. 按照1与2的规则将待探索的节点压入openset，因此openset会按照cost由大到小排列，openset中从后向前的顺序即为节点优先级的顺序；
  4. 选取openset队尾的节点作为新的探索点。


算法过程如下：

  • 从起点A开始，将起点A放入close_set；

  • 在起点A周围，不断向外扩散，按顺序放到open_set的队尾，在没遇到障碍物时，与宽度有限搜索BSF一致，即所有节点的移动cost是一样的；

  • 当遇到障碍物时，由于移动进入障碍物的cost为5，因此该节点进入openset时的cost比其他节点要大，优先级会更低；

  • 若openset中的节点有多条路径可以到达，则选择cost最低的那条路径。如下图所示，最开始，cost为8的节点的父节点为cost为3的节点；之后cost为7的节点成为了新的探索点，再次连接到了cost为8的节点，但由于本次连接的新cost为7+5，大于原cost8，因此其父节点不做更新，仍为cost为3的节点；相反若cost更小，则需要更新cost与父节点；

  • 选取openset中优先级最高的节点作为新节点，不断重复以上过程，直到探索到终点B。

2.4 tips

  • 遇到无效节点（出地图/已经在close_set中），则舍弃不要
  • 停止探索的条件有：1. 探索到了终点；2. open_set为空；3. 到达了循环最大次数

4. 关于算法

  • 搜索算法的核心其实在于如何定义open_set内元素的优先级。Dijkstra的思想是：根据不同节点之间的移动cost（即有权图），选择到达该点的最低cost，更新父节点，再以该点作为新节点继续向外扩散寻找，因此是通过cost的大小来表示搜索优先级顺序的；
  • 当所有节点之间的移动cost一致时，等同于宽度优先搜索BSF；
  • 该算法更多体现了中间点与起始点之间的cost关系，cost与终止点并无关联。


5. 一些思考

  • Dijkstra是对历史移动cost进行的优先级设定，能否根据未来信息进行优先级设定呢？
